# 秩序册生成器 - 年级与班级分开存储功能改进

## 概述
- **Summary**: 改进数据存储部分，确保年级与班级分开存储，避免将年级和班级存成"一年级1班"这样的混合格式。
- **Purpose**: 提高数据存储的规范化程度，便于后续的数据分析和查询操作。
- **Target Users**: 秩序册生成器的开发人员和最终用户。

## 目标
- 确保年级以整数形式存储，班级以文本形式存储
- 改进数据提取逻辑，从混合格式中正确分离年级和班级
- 保持现有功能的完整性，确保所有查询和生成功能正常工作

## 非目标（超出范围）
- 不修改数据库表结构
- 不改变现有的API接口
- 不影响其他功能模块的正常运行

## 背景与上下文
- 当前系统已经设计为年级和班级分开存储的结构
- 但在数据提取过程中，可能存在将年级和班级混合存储的情况
- 需要改进数据提取逻辑，确保从Excel文件中正确分离年级和班级

## 功能需求
- **FR-1**: 改进数据提取逻辑，从"一年级1班"这样的格式中正确分离年级和班级
- **FR-2**: 确保年级以整数形式存储，班级以文本形式存储
- **FR-3**: 保持现有查询和生成功能的完整性

## 非功能需求
- **NFR-1**: 数据提取逻辑应具有鲁棒性，能够处理各种格式的班级信息
- **NFR-2**: 改进后的代码应保持良好的可读性和可维护性

## 约束
- **Technical**: 保持现有的数据库表结构不变
- **Dependencies**: 依赖于现有的Excel解析和数据提取模块

## 假设
- 班级信息的格式主要为"一年级1班"、"二年级2班"等形式
- 年级信息可能以汉字数字（一、二、三）或阿拉伯数字（1、2、3）形式存在

## 验收标准

### AC-1: 数据提取逻辑改进
- **Given**: Excel文件中包含"一年级1班"格式的班级信息
- **When**: 系统提取班级信息时
- **Then**: 系统应正确分离出年级为1，班级为"1班"
- **Verification**: `programmatic`

### AC-2: 年级存储格式
- **Given**: 系统提取到年级信息
- **When**: 系统存储学生信息时
- **Then**: 年级应以整数形式存储在数据库中
- **Verification**: `programmatic`

### AC-3: 班级存储格式
- **Given**: 系统提取到班级信息
- **When**: 系统存储学生信息时
- **Then**: 班级应以文本形式存储在数据库中
- **Verification**: `programmatic`

### AC-4: 现有功能完整性
- **Given**: 系统改进后
- **When**: 执行查询或生成操作时
- **Then**: 所有现有功能应正常工作
- **Verification**: `human-judgment`
